<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Query files → StarRocks engine | SQL in query files is user-authored; executed through pymysql connection |
| Pre-baked .db files → StarRocks container | .db files baked into Docker image at build time |
| Cross-driver JOINs | Data flows between ADBC drivers via StarRocks internal federation |
| Init SQL scripts → Database containers | SQL executed at container startup; no user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Tampering | Pre-baked .db files in docker/data/ | accept | .db files are generated by a Python script from known seed data. Git-tracked files are binary but their content is deterministic. Risk: zero — no user input. |
| T-01-09 | Information Disclosure | TPC-H data in .db files | accept | TPC-H is a public benchmark dataset. Seed data is synthetic test data with no PII. |
| T-01-10 | Tampering | Query files in queries/ directory | accept | Query files are version-controlled SQL. They are executed against test databases with no production data. No injection risk — all SQL is user-authored test queries. |
| T-01-11 | Spoofing | PostgreSQL DECIMAL casting | accept | DECIMAL→DOUBLE conversion is a driver behavior, not a security concern. Floating-point tolerance checks are used in test assertions. |
</threat_model>
