  <action>
    Create `run-verify.py` at project root. Python 3.11+, uses stdlib only (argparse, subprocess, pathlib, shutil, sys, time, json, os).

    **CLI interface (argparse):**
    ```
    usage: run-verify.py [-h] [--keep | --cleanup] [--subset FILTER] [--report FILE] [--skip-rebuild] fe_deb be_deb

    StarRocks ADBC Verification Suite — ship→verify→retest loop

    positional arguments:
      fe_deb            Path to starrocks-fe .deb package
      be_deb            Path to starrocks-be .deb package

    options:
      -h, --help        show this help message and exit
      --keep            Leave containers running after tests (default)
      --cleanup         Run docker compose down after tests
      --subset FILTER   Pass filter to pytest -k (e.g., "flightsql")
      --report FILE     Write JSON report to FILE (default: reports/latest.json)
      --skip-rebuild    Skip docker compose build (reuse existing images)
    ```

    **Execution flow (implement in main()):**

    1. **Validate inputs:**
       - Check both .deb files exist and are regular files (`Path(fe_deb).is_file()`)
       - Print error and exit(1) if missing
       - Print: `✓ DEB packages found: {fe_deb}, {be_deb}`

    2. **Copy .debs to docker/:**
       - `shutil.copy2(fe_deb, "docker/starrocks-fe_latest_amd64.deb")`
       - `shutil.copy2(be_deb, "docker/starrocks-be_latest_amd64.deb")`
       - Print: `✓ DEBs copied to docker/`

    3. **Build and start containers:**
       - Build: `subprocess.run(["docker", "compose", "up", "--build", "--detach"], cwd="docker/", check=True)`
       - Skip `--build` if `args.skip_rebuild` is True
       - Print: `◆ Building and starting containers...`

    4. **Wait for healthchecks:**
       - Poll `docker compose ps --format json` in a loop
       - Parse JSON to check each service's Health status
       - Wait up to 300s (StarRocks needs ~120-180s for FE+BE startup)
       - Report each service as it becomes healthy: `  ✓ sr-main healthy (45s)`
       - If any service fails healthcheck (health: "unhealthy" or timeout), print error and exit(1)
       - Print summary: `✓ All 5 services healthy`

    5. **Run tests:**
       - Build pytest command: `["pytest", "tests/", "-v", "--json-report", f"--json-report-file={args.report}"]`
       - If `args.subset`: append `["-k", args.subset]`
       - Run with `subprocess.run(cmd, cwd=".")` — do NOT set check=True; capture returncode
       - Print: `◆ Running test suite...`
       - Capture and print pytest stdout/stderr in real-time (or after)
       - Set `test_passed = (result.returncode == 0)`

    6. **Capture logs on failure:**
       - If `test_passed` is False:
         - Run `subprocess.run(["docker", "compose", "logs", "--tail=100"], cwd="docker/", capture_output=True, text=True)`
         - Print: `── Container Logs (last 100 lines) ──`
         - Print captured log output
         - Also capture: `docker compose logs sr-main --tail=200`

    7. **Report:**
       - Print summary:
         ```
         ═══════════════════════════════════════════
          Verification Complete
         ═══════════════════════════════════════════
          DEB: {fe_deb_name}, {be_deb_name}
          Result: {'✓ PASSED' if test_passed else '✗ FAILED'}
          Report: {args.report}
          Containers: {'running' if args.keep else 'stopped'}
         ```
       - Write a JSON summary to `reports/summary.json`:
         ```json
         {
           "timestamp": "ISO-8601",
           "fe_deb": "path",
           "be_deb": "path",
           "test_result": "passed|failed",
           "subset": "filter or null",
           "containers": "running|stopped",
           "services_healthy": true,
           "report_file": "path"
         }
         ```

    8. **Cleanup (if --cleanup):**
       - `subprocess.run(["docker", "compose", "down"], cwd="docker/")`
       - Print: `✓ Containers stopped and removed`

    9. **Exit:**
       - `sys.exit(0 if test_passed else 1)`

    **Graceful handling:**
    - Catch `subprocess.CalledProcessError` during build → print error, exit(1)
    - Catch `KeyboardInterrupt` → print "Interrupted", run docker compose down if --cleanup, exit(130)
    - Catch any unexpected exception → print traceback, exit(1)

    **Add execute permission:**
    `chmod +x run-verify.py`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg .deb paths → run-verify.py | User-supplied file paths validated before copy |
| run-verify.py → Docker daemon | Subprocess calls to docker compose |
| Host filesystem → docker/ directory | .deb files copied into build context |
| Host → FlightSQL TLS cert | CA cert mounted as volume; pre-generated, self-signed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-12 | Tampering | .deb file paths from CLI args | mitigate | Validate both paths exist and are regular files before copying. Use `Path.is_file()` check. |
| T-01-13 | Denial of Service | Container healthcheck timeout | accept | 300s timeout is generous for StarRocks FE+BE startup. User can Ctrl+C. |
| T-01-14 | Information Disclosure | Container logs captured on failure | accept | Logs contain only test data (no production secrets). Captured to test report only. Logs may contain Docker-internal IPs (10.x.x.x) — acceptable for local dev. |
| T-01-15 | Elevation of Privilege | Docker socket from run-verify.py | accept | Same as Plan 01-01. Required for docker compose orchestration. |
| T-01-16 | Spoofing | Self-signed TLS certs pre-generated | accept | Self-signed certs for local testing only. No external verification chain. Acceptable. |
</threat_model>
