## Tasks

### Task 1: SF1 Data Generator
**Files:** `docker/generate-sf1-data.py` (new), `docker/data/sf1/` (new directory)
**Depends on:** None

Write a Python script that generates TPC-H SF1 CSV files.

**Implementation:**
- Pure Python 3 (stdlib only: `csv`, `random`, `gzip`, `datetime`, `os`)
- Fixed seed: `random.seed(42)` for deterministic, reproducible output
- Output directory: `docker/data/sf1/`
- File naming: `region.csv`, `nation.csv`, `supplier.csv`, `part.csv`, `partsupp.csv`, `customer.csv`, `orders.csv`, `lineitem.csv`
- Each CSV has header row matching the table schema column names
- Write gzip-compressed (`.csv.gz`) to reduce Docker build context. Option: also write uncompressed for init script simplicity.

**Data model:**
- `region` (5 rows): Fixed region names (AFRICA, AMERICA, ASIA, EUROPE, MIDDLE EAST) with generated comments
- `nation` (25 rows): 5 nations per region, FK to region table, generated names from TPC-H nation set
- `part` (200,000 rows): Auto-incrementing integer PK, generated names from adjective+noun pools, types from TPC-H type set, sizes 1-50, retail prices 900-21000, generated comments
- `supplier` (10,000 rows): Sequential PK, generated names/addresses/phone numbers, FK to nation (random), account balance -999.99 to 9999.99
- `partsupp` (800,000 rows): FK to part + FK to supplier (4 suppliers per part), generated availability + cost, comments
- `customer` (150,000 rows): Sequential PK, generated names/addresses, FK to nation, acct balance, market segment from {AUTOMOBILE, BUILDING, FURNITURE, MACHINERY, HOUSEHOLD}
- `orders` (1,500,000 rows): Sequential PK, FK to customer (1 per customer per date range), dates within 1992-1998, status with distribution (O: 0.41, F: 0.40, P: 0.19), priority from {1-URGENT, 2-HIGH, 3-MEDIUM, 4-NOT SPECIFIED, 5-LOW}
- `lineitem` (6,000,121 rows): FK to orders (1-7 lineitems per order), FK to part+supplier (from partsupp), quantity 1-50, extendedprice = quantity * part retail price (adjusted), discount 0.00-0.10, tax 0.00-0.08, ship/commit/receipt dates, returnflag with weighted distribution, linestatus F/O

**Validation:**
- After generation, print row counts and verify against spec
- Print file sizes for confirmation
- Run `cd docker && python generate-sf1-data.py` and confirm all 8 files written
- Verify `wc -l` per file matches expected row count + 1 (header)

### Task 2: SF1 Data Loading (Init Scripts + Docker Compose)
**Files:** `docker/init/postgres/02-data.sql` (rewrite), `docker/init/mysql/02-data.sql` (rewrite), `docker/docker-compose.yml` (edit)
**Depends on:** Task 1

Rewrite the backend init scripts to load SF1 CSV data instead of seed data.

**PostgreSQL init (`docker/init/postgres/02-data.sql`):**
```sql
-- Phase 2: SF1 data loading via COPY (replaces seed data)
-- Order: parent tables first (FK dependencies), then child tables

TRUNCATE region CASCADE;
TRUNCATE nation CASCADE;
TRUNCATE part CASCADE;
TRUNCATE supplier CASCADE;
TRUNCATE customer CASCADE;
TRUNCATE partsupp CASCADE;
TRUNCATE orders CASCADE;
TRUNCATE lineitem CASCADE;

\COPY region FROM '/sf1-data/region.csv' WITH (FORMAT csv, HEADER true);
\COPY nation FROM '/sf1-data/nation.csv' WITH (FORMAT csv, HEADER true);
\COPY part FROM '/sf1-data/part.csv' WITH (FORMAT csv, HEADER true);
\COPY supplier FROM '/sf1-data/supplier.csv' WITH (FORMAT csv, HEADER true);
\COPY customer FROM '/sf1-data/customer.csv' WITH (FORMAT csv, HEADER true);
\COPY partsupp FROM '/sf1-data/partsupp.csv' WITH (FORMAT csv, HEADER true);
\COPY orders FROM '/sf1-data/orders.csv' WITH (FORMAT csv, HEADER true);
\COPY lineitem FROM '/sf1-data/lineitem.csv' WITH (FORMAT csv, HEADER true);

-- Preserve non-TPC-H tables from Phase 1 (test_data, departments)
-- These are in 03-cross-join.sql or 02-data.sql kept separate
```

**MySQL init (`docker/init/mysql/02-data.sql`):**
```sql
-- Phase 2: SF1 data loading via LOAD DATA INFILE (replaces seed data)

SET FOREIGN_KEY_CHECKS = 0;

TRUNCATE region;
LOAD DATA INFILE '/sf1-data/region.csv'
INTO TABLE region FIELDS TERMINATED BY ',' OPTIONALLY ENCLOSED BY '"'
LINES TERMINATED BY '\n' IGNORE 1 ROWS;

TRUNCATE nation;
LOAD DATA INFILE '/sf1-data/nation.csv'
INTO TABLE nation FIELDS TERMINATED BY ',' OPTIONALLY ENCLOSED BY '"'
LINES TERMINATED BY '\n' IGNORE 1 ROWS;

-- ... repeat for all 8 tables

SET FOREIGN_KEY_CHECKS = 1;
```

**Docker Compose update (`docker/docker-compose.yml`):**
Add volume mount for SF1 data to both backend services:
```yaml
sr-postgres:
  volumes:
    - ./init/postgres/:/docker-entrypoint-initdb.d/:ro
    - ./data/sf1/:/sf1-data/:ro        # NEW: SF1 CSV data

sr-mysql:
  volumes:
    - ./init/mysql/:/docker-entrypoint-initdb.d/:ro
    - ./data/sf1/:/sf1-data/:ro        # NEW: SF1 CSV data
```

**Cross-join data preservation:**
- The existing `test_data` and `departments` tables (used by cross-join tests) are NOT affected by this rewrite
- Create `docker/init/postgres/03-seed.sql` with the non-TPC-H seed data preserved
- Create `docker/init/mysql/03-cross-join.sql` with preserved seed data (already exists as `03-cross-join.sql`)

**Validation:**
- After init scripts run, verify row counts via `SELECT count(*) FROM region` etc. through StarRocks
- Verify idempotency: restart containers, data should be identical (no duplicates since we TRUNCATE first)
- Verify cross-join tests still pass

### Task 3: TPC-H Query Corpus (44 SQL Files)
**Files:** `queries/postgres/03-qNN-*.sql` (22 files), `queries/mysql/03-qNN-*.sql` (22 files)
**Depends on:** Task 2 (needs SF1 data loaded to validate expected row counts)

Create all 22 TPC-H standard queries adapted for StarRocks ADBC passthrough, for both PostgreSQL and MySQL catalogs.

**File naming:** `queries/{driver}/03-q{N}-{short_description}.sql`
- N = 01 to 22 (TPC-H query number)
- Example: `queries/postgres/03-q01-pricing-summary.sql`

**Each file format:**
```sql
-- TPC-H Q01: Pricing Summary Report
-- Catalog: sr_postgres, Schema: public
-- Expected: N rows
SELECT ...
FROM sr_postgres.public.lineitem
...
```

**Query adaptation rules:**
1. Use `sr_postgres.public.table` or `sr_mysql.testdb.table` for all table references
2. Date literals: `DATE '1998-12-01'`
3. Date arithmetic: Use `DATE_SUB()` / `DATE_ADD()` instead of `INTERVAL` where StarRocks requires
4. `NOT EXISTS` subqueries → test if StarRocks supports; fall back to `LEFT ANTI JOIN` if not
5. Subqueries: Add `LIMIT` if StarRocks requires it for decorrelation
6. String concatenation: Use `CONCAT()` instead of `||`
7. `EXTRACT(YEAR FROM date)` → supported
8. `CASE WHEN ... END` → supported

**Expected row counts:**
- Fill in after hand-running each query against SF1 data and recording row counts
- Use a sandbox approach: run one backend, execute query, count rows, annotate file

**Validation:**
- `pytest tests/test_queries.py -v` runs all query files
- Verify each query returns the expected row count
- Verify zero query parse errors (StarRocks SQL dialect passes)
- Run for both PostgreSQL and MySQL

## Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| SF1 data generation takes too long for 8M rows in Python | Medium | Low — generator is one-time, not per-startup | Use efficient batch writes; maybe multiprocessing |
| PostgreSQL `COPY` fails on DECIMAL→DOUBLE mismatch | Low | Medium — query tests fail | TRUNCATE + re-import recovery; fix CSV format |
| MySQL `LOAD DATA INFILE` requires `secure_file_priv` path | Medium | High — MySQL won't load | Mount to `/var/lib/mysql-files/` (default secure path) or disable secure_file_priv in MySQL config |
| TPC-H queries use SQL features unsupported by StarRocks | Medium | Medium — some queries may need significant rewrites | Skip or adapt aggressively; document limitations |
| SF1 CSV files too large for Docker build context | Medium | Low — longer build time | `.gitignore` the CSV files; generate at build time via Dockerfile RUN |
| 8M-line CSV writing causes `disk full` or OOM | Low | High — generation fails | Write in batches of 10K rows; flush to disk incrementally |
| Init script timeout during SF1 data load | Medium | Medium — healthcheck fails before load completes | Extend MySQL/PostgreSQL healthcheck start_period; load is fast per-second but startup is longer |

---

## Design Constraints

1. SF1 data must be reproducible — fixed random seed
2. Init scripts must be idempotent — TRUNCATE + re-import
3. Query files must follow existing test_queries.py conventions (comment header, catalog-qualified table names)
4. Existing non-TPC-H test tables (test_data, departments) must be preserved
5. Must work with SF1 data in memory (CSV loading into containers uses minimal extra RAM)
6. Existing `run-verify.py` CLI must work unchanged
